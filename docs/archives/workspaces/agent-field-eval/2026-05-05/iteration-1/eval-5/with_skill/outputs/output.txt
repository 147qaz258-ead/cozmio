## Problem: Three Variables Changed at Once (Confound)

The user modified the following three variables in a single session:

| # | Variable | Type |
|---|----------|------|
| 1 | Prompt (model instructions) | prompt change |
| 2 | Memory strategy (what context is included) | context strategy change |
| 3 | Trigger threshold (sensitivity cutoff) | trigger strategy change |

**Core Issue**: When multiple variables change simultaneously, the observed effect ("it feels better") cannot be causally attributed to any individual variable. This is a textbook confound. The skill's Truth Policy states explicitly:

> "如果变量同时改了多个，需要明确说明本次无法归因。"
> (If multiple variables are changed simultaneously, attribution is impossible.)

```
variant:      new_prompt + new_memory_strategy + new_trigger_threshold
```

Because all three variables changed together, this is **not a valid controlled comparison**.

## Finding

**Cannot recommend retention or discard.** The user's feeling that "it feels better" is a valid signal, but it is not sufficient evidence to attribute causal improvement to any specific variable. The three-variable confound makes this evaluation **non-diagnostic**.

**What you should do next:**

The three variables must be isolated and tested one at a time. Here is the recommended split:

| Eval Case | Variable Under Test | Others Held Constant |
|-----------|-------------------|---------------------|
| eval-5a | Prompt only | old memory strategy + old threshold |
| eval-5b | Memory strategy only | new prompt + old threshold |
| eval-5c | Trigger threshold only | new prompt + new memory strategy |

Only after isolated tests can you determine which variable (or which combination of variables) is responsible for the perceived improvement. Without this, you risk:

- Retaining a change that introduces hidden regressions in one of the three variables
- Discarding a change that is actually the active ingredient while keeping ineffective ones
- Locking in an interaction effect that may not generalize

---

## Decision

```
decision: needs_more_cases
```

The current evaluation is confounded and cannot support a `keep`, `discard`, or `revise` decision for any of the three variables.

---

## Next Action

1. **Revert to baseline** (or a known stable state) for all three variables
2. Run three separate eval cases, isolating one variable at a time:
   - **eval-5a**: Test new prompt only, keep old memory strategy and old threshold
   - **eval-5b**: Test new memory strategy only, keep new prompt and old threshold
   - **eval-5c**: Test new trigger threshold only, keep new prompt and new memory strategy
3. For each case, capture a trace and record scores across the six dimensions
4. After three isolated evaluations, you will have the evidence needed to make a valid causal decision for each variable independently
