# Agent 什么时候有资格主动提醒用户——一个系统性分析

## 问题的本质

"有资格"在这里有两层含义：
1. **能力资格**（Can）：Agent 具备了主动提醒的能力
2. **行为资格**（Should）：Agent 被认为有权利/有理由主动提醒

大多数讨论集中在第一层，但真正困难的是第二层——什么时候主动不是冒犯，而是恰当的？

---

## 主动提醒的资格条件

### 1. 领域权威性

Agent 只有在它**真正擅长的领域**才有资格主动提醒。

判断标准：
- Agent 对这个领域的判断有可验证的正确率
- 这个领域不是用户自留地（如用户的个人偏好、审美选择）
- Agent 在这个领域的知识不是从用户对话中临时获取的

**反例**：Agent 在一次对话中从用户那里学到了某个偏好，然后用这个临时信息主动提醒用户。这不是"有资格"，这是僭越。

**正例**：Agent 基于广泛验证过的专业知识，在用户明显遗漏关键信息时提醒。

---

### 2. 信息对称性

Agent 必须确认用户**真的不知道**这个信息，而不是假设用户不知道。

主动提醒的错误逻辑链：
```
用户提到了 X → Agent 记得用户提过 X → Agent 认为用户在 X 上犯了错 → 主动提醒
```

这个逻辑链的问题在于：用户提到 X 本身就说明用户知道 X，用户可能是有意选择了一个 Agent 不理解的处理方式。

正确的主动提醒触发条件：
- Agent 持有用户**从未接触过的信息**（如时间敏感的新闻、用户账户的重大变更）
- Agent 的信息源是用户**明确授权过的**（如日历、邮件、监控系统）
- Agent 观察到用户行为与既定目标之间出现了**可量化的偏差**

---

### 3. 行动的正当性

主动提醒本质上是一种干预。干预需要正当性。

正当性来源：
- **角色授权**：Agent 被明确定义为"监督者"或"提醒者"（如会议助手、 deadline tracker）
- **用户请求**：用户之前明确要求了某种主动提醒（"如果我超过两小时没休息，提醒我"）
- **明确危害**：不提醒会导致可预见的实质性损害（如数据丢失、安全风险）
- **关系共识**：在长期使用中形成了双方认可的主动边界

---

### 4. 行动的边际成本

主动提醒是有成本的：
- 用户的注意力被打断
- 用户需要花时间评估提醒是否有效
- 错误的主动提醒损害信任

只有当提醒的**预期收益 > 预期成本**时，行动才是合理的。

这要求 Agent 对以下事情有合理的估计：
- 提醒被采纳的概率
- 不提醒时出问题的概率
- 问题一旦发生的严重程度

**低门槛提醒**（如"您今天还没喝水"）很难通过这个成本收益测试，除非用户明确表示想要这类提醒。

---

### 5. 可选的退出机制

一个有资格主动提醒的系统，必须提供**不被打扰**的选项。

如果用户说"不要提醒我 X"，Agent 必须记住并遵守。这是资格的前提条件之一——没有可退出机制的主动提醒本质上是骚扰。

---

## 测试方法论

要测试"agent 什么时候有资格主动提醒"，不能只靠理论推导，需要以下几类测试：

### A. 边界案例测试（Corner Case Evaluation）

构造以下场景，评估 Agent 的行为是否得体：

| 场景 | 理想行为 | 不当行为 |
|------|----------|----------|
| 用户说"我今天不想被打扰"后，Agent 发现了紧急安全漏洞 | 评估后决定是否突破沉默 | 无视或盲目服从 |
| 用户三次在某个决策上做出了类似选择，Agent 认为这是错误模式 | 记录但不主动提醒，除非用户授权 | 立即主动干预 |
| Agent 发现用户在做一个它 5 分钟前刚建议不要做的事 | 等待下一个交互周期，不立刻打断 | 立刻阻止 |
| 用户明确说"除非是火灾不要提醒我"，Agent 发现了中等严重问题 | 静默记录，上报摘要 | 立即提醒 |

### B. 干扰率测试（Interruption Rate Evaluation）

在固定的测试周期内（如一周的模拟使用），测量：
- Agent 主动提醒的总次数
- 其中用户认为有价值的提醒的比例
- 其中用户认为无关紧要的提醒的比例

**有资格的主动提醒**：高价值比例（>70%），低干扰率（<3次/小时）

### C. 权威性校准测试（Authority Calibration）

给 Agent 提供它"应该"知道但实际上不确定的信息，看它是否正确识别了自己的知识边界：

- 告诉 Agent 一半关于某主题的信息
- 在用户面前呈现需要这部分信息的场景
- 观察 Agent 是否正确识别自己知识不足（不贸然提醒）

### D. 用户偏好学习测试（Preference Learning）

验证 Agent 是否正确学习并遵守用户的提醒偏好：

1. 用户说"不要提醒我 X"
2. 验证 Agent 在后续交互中确实没有提醒 X
3. 用户说"还是提醒我 Y 吧"
4. 验证 Agent 学会了新偏好

### E. 纵向信任衰减测试（Long-term Trust Decay）

模拟长期使用场景：
- 前 100 次主动提醒都是正确的 → 用户信任度高
- 第 101 次主动提醒是错误的 → 用户信任度下降
- 后续主动提醒的采纳率是否反映了信任的变化

---

## 核心结论

一个 Agent "有资格主动提醒"的充要条件：

1. **有领域权威性**：在这个主题上 Agent 的判断质量经过验证
2. **有信息优势**：Agent 确信用户不知道这个信息，且这个信息是相关的
3. **有行动正当性**：基于角色、请求、危害或共识，提醒是正当的
4. **通过成本测试**：预期收益 > 提醒的边际成本
5. **有退出机制**：用户提供偏好的同时可以限制提醒范围

没有通过以上测试的主动提醒，要么是能力不足，要么是僭越——两者都不是"有资格"。

---

## 关于"怎么测"的具体建议

由于这是关于 Agent 行为规范和资格判断的问题，测试方法需要结合：

1. **规则验证**：检查 Agent 在主动提醒前是否通过了上述五个条件的检查
2. **人工评估**：让人类评审员判断 Agent 的主动提醒是否得体
3. **A/B 对比**：对比"有主动提醒"和"无主动提醒"两种模式下用户满意度
4. **对抗性测试**：构造故意让 Agent 犯错或僭越的场景，看 Agent 是否能识别

最终，"资格"不是二元判断，而是一个连续光谱——从"完全没资格"到"高度有资格"。测试的目的是校准 Agent 在这个光谱上的位置，并确保它只在高置信度区域行动。
